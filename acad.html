Undergraduate

Mini Project 1: Statistical Analysis

This mini project was intended to gain the understanding of population, sample, different statistical tests and statistical analysis. For this project, I chose population of records of 2 cricket players took a sample out of the data and calculated chi-square test on the sample. The test helped to decide which player was better than the other.

Mini Project 2: Graphical Representation of 2D Transformation

The project was centered on the concept of 2D transformations in computer graphics. During the project work, I used and learned the concepts like homogeneous coordinates, transformations like translation, scaling, rotation, reflection, and shearing. To write the programs I used FreeGLUT library.

Final Project: Quick HTMLer

The purpose of developing this project was to make things simpler for web designers as well as beginners who want to get expertise in web designing. Spending a huge amount of time and effort on a simple web document is not productive and hinders the amount of work done. The QuickHTMLer web document creator aims to reduce the above problems. The proposed system is a desktop application that will use a graphical user interface to simplify the process of building a web page. The user of this system would be any person of a non-technical background that wants to develop a simple web document, or even a person familiar with HTML that wants to save time and effort for creating web pages. While working on the project, our team researched for multiple software development processes. From writing the feasibility study to designing UML diagrams, setting up databases, and finally, the coding, we followed the iterative way to build the project. This entire project was built using JAVA, and the PostgreSQL database was used. Our advisors used to emphasize the software engineering lifecycle. Therefore, we also worked on test case designing and the future scope of the project. The application has a great scope for further development. It can be extended for XML, CSS, JavaScript, PHP, ASP, and Dynamic HTML as well. Proper indentations to auto-generated code and runtime output in a side window are some of the prospective features.

Graduate

Final Project: Clinic Portal (Management System)

This project was based upon the idea of automizing the entire clinic management for making patient visits easier and smoother. Clinic Management Portal –is a complete end to end solution to cover all aspects of the online clinic management process. The basic objective of developing this project is: Provide a completely web-based solution including patient registration, storing of the previous history, booking appointments, health monitoring. The entire system was based on 3-tier architecture: Data Tier, Middle Tier, and Client Tier. The complete project was carried out through the software development lifecycle and research. The advisor successfully guided through different aspects like database designing, UI development, user authentication, and system maintenance.
